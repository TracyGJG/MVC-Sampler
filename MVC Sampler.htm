MVC implementation of Input to Output binding.

  1. The View and Model are decoupled and only able to communicate via the Controller.
  2. Data typed by the user in sent by the View to the Controller for processing.
  3. Data sent to the Controller is forwarded to the Model for processing.
  4. Data received by the Model is processed and results sent to the Controller for presentation.
  5. The Controller sends data for presentation on to the View.
  6. The View presents processed data to the User.
